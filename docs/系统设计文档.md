## 一、系统概述

### 1.1 项目定位

鬼火论坛是一个**强互动、重玩法**的社区论坛。区别于传统 BBS，核心特色在于：

- **积分经济体系**：金币贯穿签到、发帖、点赞、红包、吹牛打赌等所有模块
- **红包玩法**：消耗金币刺激互动
- **吹牛打赌**：用户博弈的特色互动
- **勋章荣誉**：基于行为的成就系统

## 三、模块详细设计

### 3.1 积分经济体系（核心主线）

积分系统是贯穿所有模块的血脉，设计原则：

```
签到 → 加 coin（签到奖励）
发帖 → 扣 coin（发帖消耗）
被赞 → 加 coin（获赞奖励）
发红包 → 扣 coin（消耗积分）
抢红包 → 加 coin（获得积分）
吹牛赢 → 加 coin（博弈奖励）
吹牛输 → 扣 coin（博弈惩罚）
```

#### 金币变动规则

| 触发场景 | 金额 | 流水类型 |
|----------|------|----------|
| 每日签到 | +10 | WALLET_SIGN_IN |
| 发帖 | -5 | WALLET_POST |
| 被点赞 | +2 | WALLET_LIKE |
| 发红包 | -totalAmount | WALLET_RED_PACKET_SEND |
| 抢红包 | +amount | WALLET_RED_PACKET_RECEIVE |
| 吹牛赢 | +betAmount | WALLET_BOAST_WIN |
| 吹牛输 | -betAmount | WALLET_BOAST_LOSE |

#### 流水写入流程

```
任意产生金币变动的业务操作
   ↓
① 检查条件（余额、重复、权限）
   ↓
② SQL 原子更新 user_stat.coin += amount
   ↓
③ 查询更新后的余额
   ↓
④ INSERT wallet_log（amount +/−, current_balance, type, ref_id）
   ↓
⑤ 返回结果
```

> **注意**：步骤②④必须在同一事务内。先扣/加余额，再读余额快照写流水，保证数据一致性。

#### user_stat 统计字段同步规则

| 字段 | 变动时机 | 操作 |
|------|----------|------|
| post_count | 发帖成功 +1，删帖 −1 | 同步更新 |
| like_count | 被点赞 +1，被取消赞 −1 | 同步更新 |
| sign_count | 签到成功 +1 | 同步更新 |
| coin | 每次流水写入后更新 | 同步更新 |

### 3.7 红包模块（已完成）

**设计思路**：帖子内发红包，其他用户抢

#### 接口设计

| 接口 | 方法 | 状态 | 说明 |
|------|------|------|------|
| /api/redpacket/create | POST | ✅ | 发红包（扣金币，写 red_packet 表） |
| /api/redpacket/grab/{packetId} | POST | ✅ | 抢红包（拼手气/平分，加金币） |
| /api/redpacket/{packetId} | GET | ✅ | 红包详情 |
| /api/redpacket/records | GET | ✅ | 抢红包记录列表（分页） |

#### 发红包流程
```
① 校验发帖人金币 >= totalAmount
② 校验参数（totalAmount > 0, totalCount > 0, type = 1或2）
③ INSERT red_packet（remainCount = totalCount, status = 1, expireTime = 24h后）
④ SQL 原子更新 user_stat.coin -= totalAmount
⑤ 写钱包流水（-totalAmount, WALLET_RED_PACKET_SEND）
```

#### 抢红包流程（拼手气）
```
① 查询红包状态、剩余个数
② 校验：status = 1 && 未过期 && remainCount > 0 && 未抢过
③ 计算本次金额：
   - 拼手气：remainCount == 1 ? remainAmount : random(1, remainAmount / remainCount * 2)
   - 平分：totalAmount / totalCount
④ 原子扣减：UPDATE red_packet SET remain_count = remain_count - 1 WHERE remain_count > 0
⑤ INSERT red_packet_record（UNIQUE(packet_id, user_id) 防重）
⑥ 如果 remain_count == 0，UPDATE status = 2（已抢完）
⑦ SQL 原子更新 user_stat.coin += amount
⑧ 写钱包流水（+amount, WALLET_RED_PACKET_RECEIVE）
```

#### 防并发
- `UNIQUE(packet_id, user_id)` 数据库层拦截重复抢
- `UPDATE SET remain_count - 1 WHERE remain_count > 0` 原子扣减防超抢
- 整个流程在 `@Transactional` 内

### 3.8 吹牛打赌模块（已完成）

**设计思路**：发布者发起公开挑战，设置两个选项和正确答案，投入赌注金币。挑战者投入相同金额参与，答对获得赌注90%，答错失去赌注。

#### 接口设计

| 接口 | 方法 | 状态 | 说明 |
|------|------|------|------|
| /api/boast/create | POST | ✅ | 发起挑战（设置选项、正确答案、赌注金额、截止时间） |
| /api/boast/list | GET | ✅ | 挑战列表（进行中/已结束，分页） |
| /api/boast/{id} | GET | ✅ | 挑战详情 |
| /api/boast/bet | POST | ✅ | 参与挑战（选择选项，投入相同赌注） |
| /api/boast/settle/{id} | POST | ✅ | 结算（发布者或管理员操作） |

#### 发起挑战流程
```
① 校验参数（标题、选项不为空且不相同、correctOption=1或2、stakeAmount>0、deadline>当前时间）
② 校验发布者金币 >= stakeAmount
③ SQL 原子更新 user_stat.coin -= stakeAmount
④ INSERT boast（result = 0 进行中）
⑤ 写钱包流水（-stakeAmount, WALLET_BOAST_BET）
```

#### 参与挑战流程
```
① 校验挑战存在且进行中、未过截止时间
② 校验不能自己跟自己赌、选项有效、未重复下注
③ 校验挑战者金币 >= stakeAmount
④ SQL 原子更新 user_stat.coin -= stakeAmount
⑤ INSERT boast_bet（result = 0 未结算）
⑥ 写钱包流水（-stakeAmount, WALLET_BOAST_BET）
⑦ 更新 user_stat.boast_count + 1
```

#### 结算流程
```
① 校验结果参数（1=发布者赢, 2=挑战者赢）、挑战存在且未结算
② 校验权限：发布者或管理员
③ 查询所有 boast_bet 记录
④ 挑战者答对（option_type == correct_option）：
   - 挑战者获得赌注的90%：SQL原子加金币 + 写钱包流水（+stake_amount*0.9, WALLET_BOAST_WIN）
   - boast_bet.result = 1（赢）
   - user_stat.boast_win_count +1, boast_win_total += 赢取金额
   - 若本次赢取 > boast_best_win 则更新 boast_best_win
⑤ 挑战者答错（option_type != correct_option）：
   - 发布者获得挑战者的赌注：SQL原子加金币 + 写钱包流水（+amount, WALLET_BOAST_WIN）
   - boast_bet.result = 2（输）
   - 若本次亏损 > boast_worst_loss 则更新 boast_worst_loss
⑥ UPDATE boast SET result
⑦ 批量更新 boast_bet 记录
⑧ 批量写钱包流水
```

#### 排行榜

| 排行榜 | 排序字段 | 说明 |
| :--- | :--- | :--- |
| 赌神榜 | `boast_win_total` DESC | 谁累计赢的币最多 |
| 最佳一搏 | `boast_best_win` DESC | 谁一把赢的最多 |
| 最惨一亏 | `boast_worst_loss` DESC | 谁一把输的最多 |

## 四、Redis 使用设计

### 4.1 使用场景

| 场景 | Key 格式 | 数据类型 | 说明 |
|------|----------|----------|------|
| Sa-Token 会话 | `satoken:session:*` | String | 已接入 |
| 浏览量增量 | `post:views:{postId}` | String(INCR) | 每次访问 +1，定时同步 DB |
| 连续签到 | `sign:{yyyy-MM}:{userId}` | Bitmap | 查询连续签到天数 |
| 排行榜（金币） | `rank:coin` | ZSet | score = coin，ZREVRANGE 取 Top |
| 排行榜（获赞） | `rank:like` | ZSet | score = like_count |
| 红包预扣 | `redpacket:remain:{packetId}` | String(INCR) | 原子扣减剩余个数 |
| 接口限流 | `rate:{userId}:{api}` | String(INCR) | 滑动窗口限流 |

### 4.2 排行榜实现

```java
// 更新排行
redisTemplate.opsForZSet().incrementScore("rank:coin", userId, coin);

// 查 Top 10
redisTemplate.opsForZSet().reverseRangeWithScores("rank:coin", 0, 9);

// 查自己的排名
redisTemplate.opsForZSet().reverseRank("rank:coin", userId);
```
