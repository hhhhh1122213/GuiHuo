# 鬼火论坛后端接口文档

> Base URL: `http://localhost:8080`
> 认证方式: Header 携带 `token`（登录后获取）
> 更新时间：2026-05-14

---

## 一、认证模块 `/api/auth`

### 1. 注册
```
POST /api/auth/register
```
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | String | 是 | 用户名，2-20位 |
| password | String | 是 | 密码，6-32位 |

```json
// 请求
{ "username": "zhangsan", "password": "123456" }
// 响应
{ "code": 200, "msg": "success", "data": null }
```

### 2. 登录
```
POST /api/auth/login
```
```json
// 请求
{ "username": "zhangsan", "password": "123456" }
// 响应
{
  "code": 200,
  "msg": "success",
  "data": {
    "tokenName": "token",
    "tokenValue": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
  }
}
```
> 后续请求 Header 加 `token: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`

### 3. 登出
```
POST /api/auth/logout
```
> 需登录

### 4. 当前用户信息
```
GET /api/auth/info
```
> 需登录。返回 AuthInfoVO（User + UserStat 合并）

```json
{
  "code": 200,
  "data": {
    "id": 1,
    "username": "zhangsan",
    "nickname": "张三",
    "avatar": "/uploads/xxx.png",
    "role": "USER",
    "status": 1,
    "createTime": "2026-04-28T10:00:00",
    "coin": 100,
    "postCount": 5,
    "likeCount": 20,
    "signCount": 7,
    "boastCount": 3,
    "boastWinCount": 2,
    "boastWinTotal": 150
  }
}
```

### 5. 修改密码
```
POST /api/auth/password
```
> 需登录。校验旧密码正确后更新密码。

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| oldPassword | String | 是 | 旧密码 |
| newPassword | String | 是 | 新密码，6-32位 |

```json
// 请求
{ "oldPassword": "123456", "newPassword": "654321" }
// 响应
{ "code": 200, "msg": "success", "data": null }
```

```json
// 请求

```

## 三、评论模块 `/api/comments`

### 1. 评论列表
```
GET /api/comments/list?postId=1&page=1&size=20
```
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| postId | Long | - | 帖子ID，必填 |
| page | int | 1 | 页码 |
| size | int | 20 | 每页条数 |

> 分页一级评论（parentId=null），子回复挂在父评论的 `children` 字段下，不单独分页。
>
> **前端适配**：遍历 `data.records` 取一级评论，每条评论的 `children` 数组渲染楼中楼回复。`children` 为空数组时表示该评论无回复，`children[i].children` 始终为 null（只支持两层）。

```json
// 响应 - Page<CommentVO>
{
  "code": 200,
  "data": {
    "records": [
      {
        "id": 1,
        "postId": 1,
        "parentId": null,
        "content": "一楼",
        "likeCount": 3,
        "createTime": "2026-04-28T10:00:00",
        "author": { "id": 2, "nickname": "李四", "avatar": "/uploads/xxx.png" },
        "replyUser": null,
        "children": [
          {
            "id": 2,
            "postId": 1,
            "parentId": 1,
            "content": "回复一楼",
            "likeCount": 1,
            "createTime": "2026-04-28T10:05:00",
            "author": { "id": 3, "nickname": "王五", "avatar": "/uploads/xxx.png" },
            "replyUser": { "id": 2, "nickname": "李四", "avatar": "/uploads/xxx.png" },
            "children": null
          }
        ]
      }
    ],
    "total": 50,
    "current": 1,
    "size": 20
  }
}
```

### 2. 发表评论
```
POST /api/comments/create
```
> 需登录

```json
// 请求 - 直接评论帖子
{ "postId": 1, "content": "评论内容" }
// 楼中楼回复
{ "postId": 1, "parentId": 5, "replyUserId": 2, "content": "@张三 你说得对" }
```
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| postId | Long | 是 | 帖子ID |
| parentId | Long | 否 | 父评论ID，null=直接评论帖子 |
| replyUserId | Long | 否 | 被回复的用户ID，用于展示"回复 @xxx" |
| content | String | 是 | 评论内容 |

### 3. 删评论
```
DELETE /api/comments/{id}
```
> 需登录，仅作者可删。**软删除**：将 status 置为 0，防止楼中楼断裂。

## 十二、红包模块 `/api/redpacket`

> 全部需登录

### 1. 发红包
```
POST /api/redpacket/create
```
```json
// 请求
{
  "postId": 1,
  "totalAmount": 100,
  "totalCount": 10,
  "type": 1,
  "greeting": "恭喜发财"
}
```
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| postId | Long | 是 | 关联帖子ID |
| totalAmount | Long | 是 | 总金额（金币），需 > 0 |
| totalCount | Integer | 是 | 红包个数，需 > 0 |
| type | Integer | 是 | 1=拼手气红包，2=平均红包 |
| greeting | String | 否 | 祝福语 |

> 扣除发布者金币，24小时过期

### 2. 抢红包
```
POST /api/redpacket/grab/{packetId}
```
> 拼手气红包：随机金额（二倍均值法），最后一个抢的拿剩余全部。平均红包：总金额 / 总个数。
> 防重复抢：数据库唯一索引 `UNIQUE(packet_id, user_id)`，原子扣减防超抢。

```json
// 响应 - RedPacketRecord
{
  "code": 200,
  "data": {
    "id": 1,
    "packetId": 1,
    "userId": 2,
    "amount": 15,
    "createTime": "2026-04-28T10:05:00"
  }
}
```

### 3. 红包详情
```
GET /api/redpacket/{packetId}
```
```json
{
  "code": 200,
  "data": {
    "id": 1,
    "postId": 1,
    "userId": 1,
    "totalAmount": 100,
    "totalCount": 10,
    "remainAmount": 40,
    "remainCount": 4,
    "type": 1,
    "greeting": "恭喜发财",
    "status": 1,
    "expireTime": "2026-04-29T10:00:00",
    "createTime": "2026-04-28T10:00:00"
  }
}
```
| status | 说明 |
|--------|------|
| 1 | 进行中 |
| 2 | 已抢完 |
| 3 | 已过期 |

### 4. 抢红包记录
```
GET /api/redpacket/records?packetId=1&page=1&size=20
```
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| packetId | Long | - | 红包ID，必填 |
| page | int | 1 | 页码 |
| size | int | 20 | 每页条数 |

## 十三、吹牛打赌模块 `/api/boast`

> 全部需登录

### 1. 发起挑战
```
POST /api/boast/create
```
```json
// 请求
{
  "title": "今晚湖人能赢吗",
  "optionOne": "能赢",
  "optionTwo": "不能赢",
  "correctOption": 1,
  "stakeAmount": 50,
  "deadline": "2026-05-14T00:00:00"
}
```
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | String | 是 | 挑战标题 |
| optionOne | String | 是 | 选项一 |
| optionTwo | String | 是 | 选项二，不能与选项一相同 |
| correctOption | Integer | 是 | 正确答案：1=选项一，2=选项二 |
| stakeAmount | Long | 是 | 赌注金额（金币），需 > 0 |
| deadline | LocalDateTime | 是 | 截止时间，需晚于当前时间 |

> 扣除发布者赌注金币，不能自己跟自己赌

### 2. 挑战列表
```
GET /api/boast/list?page=1&size=10&status=0
```
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| page | int | 1 | 页码 |
| size | int | 10 | 每页条数 |
| status | Integer | 无 | 0=进行中，1=已结束，不传=全部 |

```json
{
  "code": 200,
  "data": {
    "records": [
      {
        "id": 1,
        "userId": 1,
        "title": "今晚湖人能赢吗",
        "image": null,
        "optionOne": "能赢",
        "optionTwo": "不能赢",
        "correctOption": 1,
        "stakeAmount": 50,
        "result": 0,
        "deadline": "2026-05-14T00:00:00",
        "createTime": "2026-05-13T10:00:00"
      }
    ],
    "total": 10,
    "current": 1,
    "size": 10
  }
}
```
| result | 说明 |
|--------|------|
| 0 | 进行中 |
| 1 | 发布者赢 |
| 2 | 挑战者赢 |

### 3. 挑战详情
```
GET /api/boast/{id}
```
> 返回 Boast 实体，结构同列表中单条记录

### 4. 参与挑战
```
POST /api/boast/bet?boastId=1&optionType=1
```
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| boastId | Long | 是 | 挑战ID |
| optionType | Integer | 是 | 选择的选项：1=选项一，2=选项二 |

> 需投入与发布者相同金额的赌注。每人每个挑战只能参与一次。不能参与自己发起的挑战。

```json
// 响应 - BoastBet
{
  "code": 200,
  "data": {
    "id": 1,
    "boastId": 1,
    "userId": 2,
    "optionType": 1,
    "amount": 50,
    "result": 0,
    "createTime": "2026-05-13T10:05:00"
  }
}
```

### 5. 结算挑战
```
POST /api/boast/settle/{id}?result=1
```
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | Path | 是 | 挑战ID |
| result | Integer | 是 | 1=发布者赢，2=挑战者赢 |

> 仅发布者或管理员可操作。已结算的挑战不可重复结算。
>
> - 挑战者答对：获得赌注的 90% 金币
> - 挑战者答错：失去赌注，赌注归发布者

```json
// 响应 - 更新后的 Boast
{
  "code": 200,
  "data": {
    "id": 1,
    "result": 1,
    ...
  }
}
```

## 通用说明

### 统一响应格式
```json
{ "code": 200, "msg": "success", "data": {} }
```
| code | 说明 |
|------|------|
| 200 | 成功 |
| 401 | 未登录 |
| 403 | 权限不足 |
| 500 | 业务异常，msg 为错误描述 |

### 软删除说明
- 帖子和评论的删除操作均为**软删除**（status 0→已删除，1→正常）
- 查询列表时自动过滤已删除记录，数据不会真正从数据库消失
- 已删除的评论不影响楼中楼的层级结构

### 认证白名单（无需登录）
- `POST /api/auth/login`
- `POST /api/auth/register`
- `GET /api/posts/list`
- `GET /api/posts/detail/**`
- `GET /api/posts/essence`
- `GET /api/posts/search`
- `GET /api/category/list`
- `GET /api/comments/list`
- `GET /api/users/*`
- `GET /api/users/*/posts`
- `GET /api/tags/list`
- `GET /api/boast/list`
- `GET /api/boast/*`

其余接口均需 Header 携带 `token`。

### 分页说明
所有分页接口的 `page` 默认为 1，`size` 默认为 20。响应中的 `total` 为总记录数，`current` 为当前页码。
