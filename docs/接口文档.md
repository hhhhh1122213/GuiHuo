# 鬼火论坛后端接口文档

> Base URL: `http://localhost:8080`
> 认证方式: Header 携带 `token`（登录后获取）

---

## 一、认证模块 `/api/auth`

### 1. 注册
```
POST /api/auth/register
```
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | String | 是 | 用户名，2-20位 |
| password | String | 是 | 密码，6-32位 |

```json
// 请求
{ "username": "zhangsan", "password": "123456" }
// 响应
{ "code": 200, "msg": "success", "data": null }
```

### 2. 登录
```
POST /api/auth/login
```
```json
// 请求
{ "username": "zhangsan", "password": "123456" }
// 响应
{
  "code": 200,
  "msg": "success",
  "data": {
    "tokenName": "token",
    "tokenValue": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
  }
}
```
> 后续请求 Header 加 `token: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`

### 3. 登出
```
POST /api/auth/logout
```
> 需登录

### 4. 当前用户信息
```
GET /api/auth/info
```
> 需登录。返回 AuthInfoVO（User + UserStat 合并）

```json
{
  "code": 200,
  "data": {
    "id": 1,
    "username": "zhangsan",
    "nickname": "张三",
    "avatar": "/uploads/xxx.png",
    "role": "USER",
    "status": 1,
    "createTime": "2026-04-28T10:00:00",
    "coin": 100,
    "postCount": 5,
    "likeCount": 20,
    "signCount": 7
  }
}
```

## 二、帖子模块 `/api/posts`

### 1. 帖子列表
```
GET /api/posts/list?page=1&size=20&categoryId=1
```
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| page | int | 1 | 页码 |
| size | int | 20 | 每页条数 |
| categoryId | Long | 无 | 分类ID，不传则查全部 |

```json
// 响应 - PostSummaryVO，仅返回 status=1 的正常帖子
{
  "code": 200,
  "data": {
    "records": [
      {
        "id": 1,
        "title": "帖子标题",
        "content": "帖子内容",
        "viewCount": 100,
        "likeCount": 10,
        "commentCount": 5,
        "isTop": false,
        "isEssence": true,
        "createTime": "2026-04-28T10:00:00",
        "categoryId": 1,
        "categoryName": "综合讨论",
        "author": {
          "id": 1,
          "nickname": "张三",
          "avatar": "/uploads/xxx.png"
        }
      }
    ],
    "total": 50,
    "current": 1,
    "size": 20
  }
}
```

### 2. 精华帖
```
GET /api/posts/essence?page=1&size=20
```

```json
    "title": "帖子标题",

    "categoryId": 1,
    "categoryName": "综合讨论",
    "author": {
      "id": 1,
      "nickname": "张三",
      "avatar": "/uploads/xxx.png"
    },
    "tags": [
      { "id": 1, "name": "Java" }
    ],
    "isLiked": false,
    "isFavorited": false
```

### 4. 发帖
```
POST /api/posts/create
```
> 需登录

```json
// 请求
{ "categoryId": 1, "title": "帖子标题", "content": "帖子内容" }
```

### 5. 搜索
```
GET /api/posts/search?keyword=关键词&page=1&size=20
```
> 按标题和内容模糊匹配，仅搜索 status=1 的帖子

### 6. 删帖
```
DELETE /api/posts/{id}
```
> 需登录，仅作者可删。**软删除**：将 status 置为 0，数据不物理删除。

## 三、评论模块 `/api/comments`

### 1. 评论列表
```
GET /api/comments/list?postId=1
```
```json
// 响应 - CommentVO，按时间升序，仅返回 status=1 的正常评论，支持楼中楼
{
  "code": 200,
  "data": [
    {
      "id": 1,
      "postId": 1,
      "parentId": null,
      "content": "一楼",
      "likeCount": 3,
      "createTime": "2026-04-28T10:00:00",
      "author": {
        "id": 2,
        "nickname": "李四",
        "avatar": "/uploads/xxx.png"
      },
      "replyUser": null
    },
    {
      "id": 2,
      "postId": 1,
      "parentId": 1,
      "content": "回复一楼",
      "likeCount": 1,
      "createTime": "2026-04-28T10:05:00",
      "author": {
        "id": 3,
        "nickname": "王五",
        "avatar": "/uploads/xxx.png"
      },
      "replyUser": {
        "id": 2,
        "nickname": "李四",
        "avatar": "/uploads/xxx.png"
      }
    }
  ]
}
```
> `parentId` 为 null = 直接评论帖子；有值 = 楼中楼回复
> `replyUser` = 被回复的用户信息，null 表示直接回复帖子

### 2. 发表评论
```
POST /api/comments/create
```
> 需登录

```json
// 请求 - 直接评论帖子
{ "postId": 1, "content": "评论内容" }
// 楼中楼回复
{ "postId": 1, "parentId": 5, "replyUserId": 2, "content": "@张三 你说得对" }
```
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| postId | Long | 是 | 帖子ID |
| parentId | Long | 否 | 父评论ID，null=直接评论帖子 |
| replyUserId | Long | 否 | 被回复的用户ID，用于展示"回复 @xxx" |
| content | String | 是 | 评论内容 |

### 3. 删评论
```
DELETE /api/comments/{id}
```
> 需登录，仅作者可删。**软删除**：将 status 置为 0，防止楼中楼断裂。

## 八、点赞模块 `/api/like`

> 全部需登录

### 1. 点赞
```
POST /api/like?targetId=1&targetType=1
```
| 参数 | 类型 | 说明 |
|------|------|------|
| targetId | Long | 目标ID |
| targetType | Integer | 1=帖子，2=评论 |

> 重复点赞会触发数据库唯一索引 `UNIQUE(user_id, target_id, target_type)` 拦截，返回"已点赞"

### 2. 取消点赞
```
DELETE /api/like?targetId=1&targetType=1
```

### 3. 是否已点赞
```
GET /api/like/check?targetId=1&targetType=1
```
```json
{ "code": 200, "data": true }
```

## 通用说明

### 统一响应格式
```json
{ "code": 200, "msg": "success", "data": {} }
```
| code | 说明 |
|------|------|
| 200 | 成功 |
| 401 | 未登录 |
| 403 | 权限不足 |
| 500 | 业务异常，msg 为错误描述 |

### 软删除说明
- 帖子和评论的删除操作均为**软删除**（status 0→已删除，1→正常）
- 查询列表时自动过滤已删除记录，数据不会真正从数据库消失
- 已删除的评论不影响楼中楼的层级结构

### 认证白名单（无需登录）
- `POST /api/auth/login`
- `POST /api/auth/register`
- `GET /api/posts/list`
- `GET /api/posts/detail/**`
- `GET /api/posts/essence`
- `GET /api/posts/search`
- `GET /api/category/list`
- `GET /api/comments/list`
- `GET /api/users/*`
- `GET /api/tags/list`

其余接口均需 Header 携带 `token`。

### 分页说明
所有分页接口的 `page` 默认为 1，`size` 默认为 20。响应中的 `total` 为总记录数，`current` 为当前页码。
